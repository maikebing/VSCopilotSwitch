# Agent Guidelines

本文件是当前仓库的智能体协作要求。所有自动化智能体、Codex 会话和后续贡献者都应遵守这些规则。

## 项目定位

VSCopilotSwitch 的目标是把多个模型供应商统一转换为 Ollama 兼容协议，并自动维护 VS Code 用户配置。任何实现都应优先保护用户配置、安全凭据和跨平台可用性。

## 工作原则

- 优先保持用户数据安全，不得静默删除、覆盖或重置用户配置。
- 修改 VS Code 配置前必须设计备份、差异预览和回滚路径。
- 所有写入操作应幂等，多次执行不能产生重复条目或配置漂移。
- API Key、Authorization Header、Cookie、代理密码等敏感信息不得写入普通日志。
- 不要把供应商私有协议硬编码到核心层，应通过 Provider Adapter 隔离。
- 跨平台代码必须考虑 Windows、macOS、Linux、WSL 的路径、权限、托盘能力和后台进程差异。
- UI 需要降低误操作风险，关键写入动作应让用户能理解将修改什么。

## 推荐架构边界

建议保持以下模块职责：

| 模块 | 职责 |
| --- | --- |
| host | OmniHost 宿主、窗口、托盘、后台生命周期、SPA 嵌入式资源加载 |
| core | 路由、协议模型、熔断、重试、配置数据结构 |
| providers | 各模型供应商协议适配器 |
| vscode-config | VS Code 配置发现、读取、备份、写入、回滚 |
| ui | Vue 3 SPA、供应商配置、模型切换、状态面板、日志和向导 |
| tests | 单元测试、集成测试和配置写入快照测试 |

## Provider Adapter 要求

新增供应商时，需要至少包含：

- 供应商名称、基础 URL、鉴权方式和模型命名规则。
- 模型列表获取或静态模型声明能力。
- Ollama chat 请求到上游请求的转换。
- 上游非流式响应到 Ollama 响应的转换。
- 上游流式响应到 Ollama 流式响应的转换。
- 错误结构映射，不直接暴露敏感请求信息。
- 超时、重试、熔断参与方式。
- 最小测试样例或协议快照。

## VS Code 配置修改要求

处理 `settings.json` 和 `chatLanguageModels.json` 时必须遵守：

- 写入前读取现有内容并创建备份。
- 保留未知字段、注释兼容策略和用户自定义配置。
- 写入只触及本项目管理的 Ollama 相关字段。
- 失败时返回清晰错误并保留原文件。
- 路径解析不能假设固定用户名，示例路径只能用于文档和测试。
- WSL 下必须明确 Windows 侧 VS Code 配置和 Linux 侧配置的差异。

## 安全要求

- API Key 只能存储在受保护的本地存储中，后续优先使用系统 Keychain / Credential Manager / Secret Service。
- 日志中只允许出现脱敏后的密钥，例如 `sk-...abcd`。
- 导出配置默认不包含密钥，除非用户明确选择包含并确认风险。
- 本地代理默认只监听 `127.0.0.1`，不得默认暴露到局域网。
- 如果未来支持远程访问，必须增加鉴权和明确的风险提示。

## UI 要求

- UI 参考 `cc switch` 的快速切换体验，但不要简单复制。
- 首页应突出当前生效模型、供应商、健康状态和 VS Code 配置状态。
- 关键按钮文案要具体，例如“写入 VS Code Ollama 配置”，不要只写“应用”。
- 失败状态应给出可执行的修复建议。
- 高级能力如代理、熔断阈值、重试次数应默认折叠。
- UI 必须使用 Vue 3 SPA 组织页面和路由。
- 开发阶段应通过项目内 npm 脚本启动和调试 SPA，不依赖仓库外部手工步骤。
- 发布阶段必须支持将 SPA 静态产物作为嵌入式资源打包进单体应用。
- 正式包运行时应优先加载嵌入式 SPA 资源，开发模式才允许连接 npm 调试服务。
- 托盘图标必须支持打开或聚焦主界面、查看当前提供商、快速切换提供商和退出程序。


## 构建与发布要求

- 前端依赖、调试和构建应通过明确 npm scripts 管理。
- SPA 构建产物目录必须稳定，方便宿主项目在发布时收集。
- AOT 发布前需要验证嵌入式资源读取、托盘菜单、配置读写和本地代理启动。
- 单体应用发布包不能依赖用户机器安装 npm、Node.js 或前端开发服务器。
- 如果开发模式支持连接 npm 调试服务，必须通过显式开发配置开启，不能在正式包默认启用。

## 测试要求

在实现对应功能时，优先补齐以下测试：

- Provider 请求/响应映射快照测试。
- VS Code 配置读写幂等测试。
- 配置写入失败回滚测试。
- 熔断状态转换测试。
- 跨平台路径解析测试。
- 流式响应转换测试。
- Vue 3 SPA 构建产物嵌入式资源加载测试。
- 托盘菜单行为测试或最小人工验收清单。
- AOT 单体发布冒烟测试。

## 文档维护

- 用户可见行为变化需要更新 `README.md`。
- 阶段性计划变化需要更新 `ROADMAP.md`。
- 每次有用户可感知变化时更新 `CHANGELOG.md`。
- 新增架构约束或协作要求时更新本文件。
